# req_res テスト仕様

## 概要

API リクエスト / レスポンス構造体の JSON シリアライゼーション往復テスト。全構造体が正しく marshal / unmarshal されることを検証する。

## テストフレームワーク

Go `testing` パッケージ

## テストファイル

| ファイル | テスト内容 |
|---------|-----------|
| `req_res_test.go` | 16種のリクエスト/レスポンス構造体の JSON 往復テスト |

## テスト内容

- **JSON Roundtrip**: 構造体 → JSON → 構造体 の往復変換で値が保持されることを検証
- **対象**: 全 API エンドポイントで使用される主要なリクエスト/レスポンス型（16種）
- **検証項目**: フィールド名のマッピング、null/空配列の扱い、ネストされた構造体の正確な変換

## 実行方法

```bash
cd src/server && go test ./gkill/api/req_res/...
```

または:

```bash
npm run test_server
```
