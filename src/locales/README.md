# locales - 国際化（i18n）

## 概要

vue-i18n 用の翻訳ファイル群。7言語に対応し、フロントエンドの全 UI テキストを管理する。
主言語は日本語（`ja.json`）。

## ファイル一覧

| ファイル | 言語 | 説明 |
|---------|------|------|
| `ja.json` | 日本語 | **主言語**。全キーの定義元 |
| `en.json` | 英語 | |
| `zh.json` | 中国語 | |
| `ko.json` | 韓国語 | |
| `es.json` | スペイン語 | |
| `fr.json` | フランス語 | |
| `de.json` | ドイツ語 | |

## 連携

### フロントエンドとの連携

`src/client/i18n.ts` でこれらの JSON ファイルが読み込まれ、vue-i18n に登録される。
コンポーネント内では `$t('key')` または `t('key')` で翻訳テキストを取得。

### ビルドとの連携

`npm run install_server` / `npm run install_app` のビルドパイプラインで、
`src/locales/` のファイルが `src/server/gkill/api/embed/` 配下にコピーされ、
Go バイナリに埋め込まれる。

### サーバ側との連携

Go バックエンドも `go-i18n/v2` でサーバ側メッセージの国際化を行う。
API レスポンスのエラーメッセージ・通知メッセージに使用。

## 開発ガイドライン

### 新しい翻訳キーの追加

1. `ja.json` に新しいキーと日本語テキストを追加
2. 他の6言語ファイルにも同じキーで翻訳テキストを追加
3. `ja.json` が正規の定義元なので、まず日本語で追加してから他言語に展開

### キーの命名規則

既存のキー構造に従う。フラットなキー構造（ネストなし）を基本とする。

### KFTL プレフィックス

KFTL のプレフィックス文字（`。`, `ーー`, `？`, `、` 等）は i18n キーとして定義されており、
ロケールによって変更可能な設計になっている。
