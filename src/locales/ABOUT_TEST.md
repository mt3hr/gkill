# locales テスト仕様

## 概要

7言語の i18n 翻訳ファイルの完全性を検証するテスト。全ロケール間でキーの欠落や不一致がないことを保証する。

## テストフレームワーク

Vitest

## テストファイル

| ファイル | テスト内容 |
|---------|-----------|
| `src/client/__tests__/unit/i18n-completeness.test.ts` | 全7ロケールのキー完全性検証 |

## テスト内容

- **対象ロケール**: ja（日本語）, en（英語）, zh（中国語）, ko（韓国語）, es（スペイン語）, fr（フランス語）, de（ドイツ語）
- **検証項目**:
  - 基準ロケール（ja）に存在するキーが他の全ロケールにも存在すること
  - 各ロケールのキー数が一致すること（約758キー）
  - 空文字列や未翻訳のプレースホルダがないこと

## 備考

テストファイルはフロントエンド (`src/client/__tests__/`) に配置されているが、翻訳ファイル自体 (`src/locales/`) はフロントエンドとバックエンド双方で共有されている。

## 実行方法

```bash
npm run test_client_unit
```
