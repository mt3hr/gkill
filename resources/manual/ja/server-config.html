サーバ設定

サーバの動作に関する設定を行う画面です。管理者のみアクセスできます。

上部ボタン

アカウント管理ボタンを押すと、アカウント管理画面が開きます。

下部ボタン

適用: 設定を保存します。
キャンセル: 変更を破棄して画面を閉じます。

設定の変更を反映するには、サーバの再起動が必要です。

記録保管場所の管理

記録保管場所（リポジトリ）は、gkillが記録データを保存するディレクトリや場所のことです。用途（テキストメモ用・ファイル用・GPSログ用など）や端末ごとに複数の記録保管場所を設定できます。

記録保管場所の設定は、アカウント管理画面の「保存先割当ボタン」から各ユーザに対して行います。

記録保管場所の追加

  1. アカウント管理画面を開き、対象ユーザの「保存先割当ボタン」を押します。
  2. 「追加」ボタンを押して新しい記録保管場所を追加します。
  3. 以下の項目を設定します：
    • 表示名: ライフログビューやフィルターに表示される名前
    • 種別: 保存するデータの種類（テキストメモ・打刻・タスク・ファイル管理・GPSログ・Gitコミットログ等）
    • パス: データを保存するディレクトリのパス（ファイル管理の場合はglob形式も使用可）
    • 書き込み先として使用: 新しく記録を追加したときにこの記録保管場所に保存するかどうか
    • 有効: この記録保管場所を検索・表示対象とするかどうか
  4. 「保存」ボタンを押して設定を反映します。

種別の選択

種別用途
テキストメモテキストメモを保存します
打刻時間計測（打刻）の記録を保存します
タスクタスク管理の記録を保存します
支出支出の記録を保存します
ブックマークURLブックマークを保存します
気分気分値の記録を保存します
数値記録数値データを保存します
ファイル管理アップロードファイル（画像・動画等）を保存します
GPSログGPXファイル形式のGPSログを管理します
GitコミットログGitリポジトリのコミット履歴を読み取ります（読み取り専用）
ヒント: 同じ種別の記録保管場所を複数追加できます。例えば、端末ごとにテキストメモ用の記録保管場所を分けることで、端末ごとのデータを整理できます。
ヒント: 「書き込み先として使用」をオンにした記録保管場所が、新規記録の保存先になります。同じ種別で複数の記録保管場所がある場合は、書き込み先として使用する1つだけをオンにしてください。
