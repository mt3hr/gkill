AI連携

Claude などの AI アシスタントと gkill を連携することで、会話形式でライフログを検索・追加・編集できるようになります。

できること

操作例
記録の検索「先週のランチ記録を見せて」「今月の支出を集計して」
記録の追加「今日のメモを追加して」「タスクを作って」
記録の更新・削除「このタスクを完了にして」
GPS・ファイルの取得「今日の移動経路を教えて」
設定の確認「テンプレート一覧を見せて」

連携の種類

種類説明
読み取り専用AI が記録の閲覧・検索のみ行える。記録の追加・変更はできない
書き込み専用AI が記録の追加・更新・削除のみ行える
読み書き両用AI が記録の閲覧・追加・更新・削除をすべて行える

設定方法（Claude Desktop）

Claude Desktop の設定ファイル（claude_desktop_config.json）に以下を追記します。

読み取り専用で連携する場合

{
  "mcpServers": {
    "gkill": {
      "command": "npx",
      "args": ["-y", "gkill-mcp-server"],
      "env": {
        "GKILL_BASE_URL": "http://localhost:9999",
        "GKILL_USER": "あなたのユーザーID",
        "GKILL_PASSWORD_SHA256": "パスワードのSHA256ハッシュ"
      }
    }
  }
}
ヒント: GKILL_PASSWORD_SHA256 はパスワードをSHA256でハッシュ化した値です。コマンドプロンプト（Windows）で certutil -hashfile - SHA256 などのツールで生成できます。

設定方法（Claude Code）

ターミナルで以下のコマンドを実行します。

# 読み取り専用で追加
npx @anthropic-ai/claude-code mcp add gkill \
  --command npx \
  --args "-y,gkill-mcp-server" \
  --env GKILL_BASE_URL=http://localhost:9999 \
  --env GKILL_USER=ユーザーID \
  --env GKILL_PASSWORD_SHA256=ハッシュ値

リモートアクセス（外出先から利用する場合）

gkill サーバをインターネット経由でアクセスできる状態にした上で、GKILL_BASE_URL にそのアドレスを指定します。セキュリティのため TLS（HTTPS）を有効化することを推奨します。

ヒント: TLS の設定はサーバ設定から行えます。
