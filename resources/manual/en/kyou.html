Record View

The Record view displays individual record details. A record is the core entity in gkill, representing any recorded life event. Each record has a type, timestamp, tags, and type-specific data.

Record Types

TypeDescription
Memo (kmemo)Free-form text notes and memos.
Bookmark (urlog)Saved URLs with title and optional notes.
Task (mi)Tasks with board assignment, status (open/checked/completed), and limit date.
Mood (lantana)Mood values on a scale of 0 to 10.
Expense (nlog)Financial records with amount and category.
Timestamp (timeis)Time tracking sessions with start and optional end time.
File (idf_kyou)Uploaded files and images with thumbnails.
Git Commit (git_commit_log)Git commit logs pulled from monitored repositories.
Numeric (kc)Arbitrary numeric counter values.
Repost (re_kyou)A reference (repost) of another record at a different time.

Record Details

Each record view shows:

  • Timestamp: When the event occurred.
  • Tags: Associated tags displayed as chips. Tags help with filtering and organization.
  • Related Timestamps: Any linked time tracking sessions (Time Log) associated with this record.
  • Repository Name: The data repository where this record is stored.
  • Type-specific content: The main data of the record (text, URL, amount, etc.).

Context Menu

Right-click (or long-press on mobile) on a record to access the context menu with options such as:

  • Edit: Open the record for editing.
  • Copy: Copy record information to the clipboard.
  • Add Tag: Attach a new tag to the record.
  • Repost: Create a repost of this record at the current time.
  • Delete: Remove the record (moves to trash).
  • Open in new tab: View the record detail in a separate browser tab.
Tip: Tags are a powerful way to organize your records across types. You can filter by tags in the Life Log View and Tasks views to find related records quickly.
