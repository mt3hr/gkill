Aggregation View

A screen for aggregating and visualizing life log data based on conditions. You can define two types: "Items" for single aggregate values and "Lists" for grouped aggregations by key.

Tip: The Aggregation View is a toggle feature within the LifeLog View screen. Click the aggregation icon in the LifeLog View toolbar to display it.

Definition Management

Element Description
Tabs Each aggregation definition is displayed as a tab. Click a tab to switch between definitions.
Definition Name A text field where you can edit the name of the aggregation definition.
+ Button (next to tabs) Adds a new aggregation definition.
Delete Button Deletes the currently displayed aggregation definition.

Screen Elements

Element Description
Date Range Displays the time period for aggregation.
Download Button Downloads the aggregation results in JSON format.
+ Button (floating) Displays a menu to choose between adding an Item or a List.

Items (Single Aggregate Value)

Displays aggregate values of data matching the conditions in the format: "Title: Prefix [Aggregate Result] Suffix".

Setting Type Description
Title Text Input Sets the display name of the item.
Prefix Text Input Text displayed before the aggregate value.
Suffix Text Input Text displayed after the aggregate value.
Aggregate Target Dropdown Selects the aggregation method (count, sum, average, max, min, etc.).
Predicate Group Condition Editor Filters the aggregation targets using AND/OR conditions.

Lists (Grouped Aggregation)

Displays aggregation results grouped by key in a list format.

Setting Type Description
Title / Prefix / Suffix Text Input Same settings as Items.
Aggregate Target Dropdown Selects the aggregation method.
Group Key Dropdown Selects the grouping criteria (title, data type, tag, month, day of week, date, etc.).
Predicate Group Condition Editor Filters the aggregation targets using AND/OR conditions.

Predicate Group Editing

Element Description
Logic Select AND / OR / NOT to set how conditions are combined.
Add Predicate Adds a new condition to the group.
Add Group Adds a nested predicate group.
Delete Group Deletes this predicate group.
Each Condition Sets the condition type and value (filter by data type, tag, title, amount, mood value, etc.).

Operations

Operation Description
Double-click Opens the editing screen for the item or list.
Right-click Displays the context menu (Edit / Delete).
Drag & Drop Changes the display order of items and lists.

Bottom Action Buttons

Button Description
Apply Saves all definition changes.
Cancel Discards changes and closes the dialog.
Tip: Items are useful for single aggregate values (e.g. monthly total expenses), while Lists are useful for grouped aggregations (e.g. expenses by store).
Tip: Combine conditions to create complex aggregation queries.
Tip: Use drag & drop to freely reorder the display of items and lists.
Tip: Aggregation results can be downloaded as JSON for analysis with external tools.
