Tutorial

This tutorial explains how to use gkill, divided into a Basics section and an Advanced section. If you are new to gkill, please read through the Basics section first.

About this dialog: This help dialog allows mouse clicks and touch events to pass through to the screen behind it. You can interact with the app while keeping this dialog open.
Using on smartphones: gkill works as a PWA (Progressive Web App). Select "Add to Home Screen" (iOS Safari) or "Install App" (Android Chrome) from your browser menu to launch it directly from your home screen like a native app.

Terminology

In gkill, data types are called "record types" (Memo, Time Stamp, Task, etc.) and the screens where you operate are called "screen names" (Notepad, Timestamp Notepad, Task, etc.). For example, "Memo" refers to the type of text memo data, while "Notepad" refers to the screen used to input that data. Items shown in the "+" button menu such as "Notepad" and "Time Stamp" are screen names for adding each type of record.

Switching Between Screens

Tap the page title displayed in the toolbar at the top of the screen (e.g., "Life Log View", "Task", etc.) to open a dropdown menu and navigate to each screen.

Basics

1. Logging In to gkill

First Launch

When you launch gkill for the first time, the account registration screen will appear automatically.

  1. Enter a user ID (this will be your account name).
  2. Enter a password, then enter it again for confirmation.
  3. Enter an Admin Password and confirm it (this sets the password for the auto-created admin account, used for server management).
  4. Press the "Register" button to create your account.
  5. You will be redirected to the login screen. Log in with the account you just created.

Regular Login

  1. Enter your account user ID in the user ID field.
  2. Enter your password in the password field.
  3. Click the "Login" button or press the Enter key.
Tip: Your login session is valid for 30 days. You do not need to log in again until it expires.

2. Adding Records

gkill allows you to record various types of life logs. Add records using the "+" button at the bottom right of the screen.

Record TypePurposeExample
MemoSave a text memo"The weather was nice today"
Time StampRecord time (start/end)"Meeting 10:00-11:30"
TaskManage things to do"Write the report"
ExpenseRecord spending"Convenience store: rice ball 150 yen"
MoodRecord mood as a value from 0 to 10"7"
Numeric RecordsRecord numeric data"Weight: 65.5"
BookmarkSave a URL"https://example.com"
FileUpload images or filesPhotos, PDFs, etc.

How to Add a Record

  1. Tap the "+" button at the bottom right of the screen.
  2. Select the type of record you want to add.
  3. Fill in the input form with the content.
  4. Press the "Save" button to save the record.
Tip: Using the Notepad screen, you can enter multiple records at once as text. See "3. Entering Multiple Records at Once with Notepad" below for details.

3. Entering Multiple Records at Once with Notepad

On the Notepad screen, you can create multiple records at once simply by writing text.

Basic Usage

  1. Open the Notepad screen (select "Notepad" from the "+" button menu).
  2. Enter the content you want to record in the text area.
  3. Press the "Save" button (you can also type ! and submit).

Entering with Templates

Using templates, you can insert boilerplate text with a single tap.

  1. On the Notepad screen, select a template from the template list.
  2. The template content will be inserted into the text area.
  3. Edit the content as needed, then save.
Tip: It is convenient to pre-register frequently used record patterns as templates. You can manage them from "Template Structure" in the settings screen.

4. Viewing and Searching Records

On the Life Log View screen, you can display and search past records.

Basic Operations

  1. Open the Life Log View screen from the navigation at the top of the screen.
  2. Records are displayed in chronological order.
  3. Change the date to view records for a specific day.

Search and Filtering

  • Filter by tag: Display only records with a specific tag.
  • Date range: Narrow down the period by specifying a start and end date.
  • Text search: Search for records containing specific keywords.
Tip: Reducing the "Life Log View display days" setting will speed up loading.

5. Managing Tasks

On the Task screen, you can manage tasks in a Kanban board format.

Creating a Task

  1. Open the Task screen.
  2. Add a task from the "+" button.
  3. Enter the title, board name, deadline, etc.
  4. Once saved, the task will appear on the specified board.

Completing a Task

  1. Tap a completed task to open its details.
  2. Add a checkmark to mark the task as complete.
Tip: Creating multiple boards for categories like "Work," "Personal," and "Shopping" makes it easier to stay organized.

6. Changing Settings

You can customize the application behavior from the settings screen.

How to Open Settings

  1. Tap the gear icon in the toolbar at the top of the screen.
  2. The settings screen will open.

Common Settings

8. Using Templates Effectively

By saving frequently used Notepad syntax patterns as templates, you can save yourself the trouble of typing them every time.

Managing Templates

  1. Open the settings screen.
  2. Click the "Template Structure" button.
  3. You can add, edit, and delete templates.

Example of Creating a Template

For example, to create a daily diary template:

.diary
,
-ra

Using this template, a diary framework is inserted with a single tap on the Notepad screen, and you only need to add the memo content and mood value to complete the record.

Tip: Using folders to organize templates by category makes them easier to find.

9. Recording Time with Time Stamps

Screen Name Purpose
Stamp Notepad Enter memos with timestamps. File uploads are also possible.
In Progress List of currently active time tracking sessions. You can end sessions here.
Notepad Start and end time tracking via text using notepad syntax. Can be combined with other records in bulk.

Using the time stamp feature, you can accurately record work time and activity durations. "Stamp Notepad" is a dedicated screen for memo input combined with active session management, while "Notepad" lets you start/end time tracking via prefixes alongside other records in bulk.

Basic Time Stamp Operations

  1. Start: Use the -ta prefix in Notepad, or add a time stamp from the "+" menu. Specify a title and start time.
  2. End: Use the -e prefix in Notepad, or enter the end time in the time stamp edit screen. The time stamp with the matching title will be ended.

Managing Active Time Stamps

On the time stamp list screen, you can check currently active time stamps.

  • Active time stamps are displayed in a list.
  • Tap to set the end time.
Tip: Using the Notepad syntax -ie, no error will occur even if there is no matching time stamp. You can safely end without existence checking.

10. Analyzing Data with the Aggregation View

The Aggregation View within the LifeLog View screen lets you visualize recorded data with summary tables.

How to Use the Aggregation View

  1. Click the aggregation icon in the LifeLog View toolbar to show the Aggregation View.
  2. Select the aggregation type and period.
  3. Data will be displayed in summary tables.

Aggregation Settings

You can manage aggregation methods and view configurations from the "Aggregation View" button in the settings screen.

Tip: Viewing the trends of mood values and expense data over time can reveal patterns in your daily life.

11. Using External Integrations

Recording URLs with Bookmarklets

By registering a bookmarklet in your browser's bookmark bar, you can save the web page you are viewing to gkill with a single click.

  1. Copy the URL displayed in "Bookmarklet" in the settings screen.
  2. Register it as a new bookmark in your browser's bookmark bar.
  3. When viewing a web page you want to save, click the registered bookmark.

GPS Recording and Map Display

If you are recording GPS logs, you can display your movement history on a map.

  1. Set the "Google Map API Key" in the settings screen.
  2. If GPS data has been recorded, the map display will be available in the Life Log View.
Tip: To obtain a Google Map API key, enable the Maps JavaScript API in the Google Cloud Console.

12. Sharing Records

In gkill, you can generate share links to share records with others.

How to Generate a Share Link

  1. Open the detail view or context menu for the record you want to share.
  2. Select "Share."
  3. A share URL will be generated. Use the appropriate URL for your purpose:
URL TypePurpose
Local URLAccess from the same device (localhost)
LAN URLAccess from other devices on the same network
WAN URLExternal access via the internet

Shared Data Control

When sharing, you can choose which data to include.

OptionDescription
TagsInclude tags attached to the record
TextInclude the text content of the record
Time StampsInclude time stamp information
LocationInclude GPS location information
Time OnlyInclude only the time information of the record
Tip: To use LAN or WAN URLs, you need to configure the server address in the server settings.

For Recipients of Shared Links

Opening a shared link lets you view records without a gkill account.

  • The range of displayed records is determined by the link creator's settings.
  • You cannot edit or delete records (view only).
  • If the link is no longer valid, please contact the sender.

13. Data Backup and Management

Server Settings (For Administrators)

When logged in with an administrator account, a "Server Settings" button will appear in the settings screen.

  • Port Number: Change the port the server listens on (default: 9999).
  • TLS Settings: Enable HTTPS communication.
  • Access Restriction: Restrict access to local only.

Account Management

Administrators can add, enable, and disable user accounts on the "Account Management" screen.

Data Backup

All gkill data is stored in the home directory (default: ~/gkill). Backing up this directory will preserve all your data.

Tip: It is recommended to back up the home directory regularly. The datas/ and configs/ directories are particularly important.
