Memo Pad Syntax Reference

Memo Pad Syntax is a text-based recording format used in gkill. By using line prefixes, you can input various types of data at once. Below are all 18 statement types.

Statement Details

1. Memo (no prefix)

Lines without any prefix become memo text. Multiple consecutive lines are combined into one memo.

The weather was nice today
I went for a walk

2. Tag: #

Prefix with # to specify tags. Separate multiple tags with ,.

#diary,weather

3. Text Block: --

Start and end with --. Everything between is preserved as-is.

--
This is a
multiline text block
--

4. Separator: ,

Splits into a new record. Everything after this line belongs to a new record.

First memo
,
Second memo

5. Separator (+1 second): ,,

Same as separator but increments the timestamp by 1 second.

First memo
,,
Memo one second later

6. Related Time: ?

Sets a related time. Format: ?YYYY-MM-DD, ?YYYY-MM-DD HH:MM, or ?YYYY-MM-DD HH:MM:SS

?2025-03-15 14:30
Note: # at the start of a line is interpreted as a tag prefix, and ? as a date/time. To include these characters as literal text, wrap them in a text block (--).

7. Numeric Record: /num

Next line: title, then next line: numeric value.

/num
Weight
65.5

8. Task: /mi

Following lines: title, board name, deadline, estimated start time, estimated end time (all optional after title).

/mi
Write report
Work
2025-03-20

9. Mood: /mood

Next line: mood value (integer 0-10).

/mood
7

10. Expense: /expense

Next line: shop name, then pairs of (item name, amount) repeating.

/expense
Convenience store
Rice ball
150
Tea
120

11. Time Log (Full): /timeis

Next lines: title, start time, end time.

/timeis
Meeting
2025-03-15 10:00
2025-03-15 11:30

12. Time Log Start: /start

Next lines: title, start time.

/start
Start work
2025-03-15 09:00

13. Time Log End: /end

Next line: title. Finds a matching started time log and ends it.

/end
Start work

14. Time Log End (if exists): /end?

Same as Time Log End but does not produce an error if no matching time log exists.

/end?
Start work

15. End Time Log by Tag: /endt

Next line: tag name. Finds and ends a time log by its tag.

/endt
work

16. End Time Log by Tag (if exists): /endt?

Same as End Time Log by Tag but does not produce an error if no matching time log exists.

/endt?
work

17. Bookmark: /url

Next line: URL, then next line: title.

/url
https://example.com
Favorite site

18. Save: !

Terminates the document and triggers the save process.

!
