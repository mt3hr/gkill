Settings (Application Config)

This screen manages application-wide settings and data structure configuration. You can adjust display preferences, behavior options, and external integrations.

Top Action Buttons

Button Description
Server Settings (admin only) Opens the server configuration screen for port, TLS, access restrictions, and other server-level settings. Only visible to administrator accounts.
Reload Refreshes data from the server. Long-press to force a full reload.
Logout Ends the current session and logs out. Long-press for additional options.

General Settings

Setting Type Description
Language Dropdown Choose the interface language: ja (Japanese), en (English), zh (Chinese), ko (Korean), es (Spanish), fr (French), de (German).
Dark Theme Checkbox Toggle dark mode on or off. Suitable for low-light environments.
Hot Reload Checkbox When enabled, the life log view automatically refreshes when data changes are detected.
Show Tags in List Checkbox Controls whether tags are displayed on record cards in list views.
Show Share Footer Checkbox Controls whether sharing options are displayed in the footer area.

Display Settings

Setting Type Description
Life Log View Display Period Checkbox + Number Sets the number of days of records to display in the life log view. Set to -1 for unlimited. A shorter period loads faster.
Default Page Dropdown Choose which page is shown after login.
Life Log View Image Columns Number (1-10) Sets the number of columns for the thumbnail grid in the life log view. Adjust based on your screen size.
Default Task Board Dropdown + Add button Sets the default board shown on the task screen. Use the add button to create new boards.

External Integration

Setting Type Description
Bookmarklet Address Read-only A JavaScript bookmarklet URL for your browser's bookmark bar. Use it to save bookmarks to gkill with one click from any webpage.
Google Maps API Key Text input Enter your Google Maps API key to enable GPS map features. The map view requires this key to function.

Data Structure Management

These buttons open dedicated editors for managing how your data is organized.

Button Description
Tag Structure Manage the tag hierarchy and tag groupings. Create, rename, reorder, and organize tags into parent-child relationships.
Repository Structure Manage data storage locations and repositories. Add, remove, or configure where records are stored.
Profile Structure Manage device and profile configurations. Edit per-device settings.
Record Type Structure Manage which record types are stored in which repositories.
Template Structure Manage memo pad text templates for quick input. Create and edit boilerplate text entries.
Summary View Manage data aggregation and summary view configurations.
Related Information Manage related information structures.

Bottom Action Buttons

Button Description
Apply Save all changes.
Cancel Discard changes and close the screen.

Footer

The bottom of the screen displays application information:

  • Version number — Clickable link to the project page.
  • Build timestamp — Date and time when the application was built.
  • Commit hash — The Git commit the build was created from.
Tip: After changing settings, press "Apply" to save. Press "Cancel" to discard all changes.
Tip: For smartphones, set image columns to 1-2. For desktop screens, 3-5 columns works well.
Tip: To use GPS map features, enable the Maps JavaScript API in Google Cloud Console and enter the API key in External Integration.
