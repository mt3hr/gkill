Related Information

A screen for defining conditions to automatically search and display records related to a life log entry. You can set up queries per definition to visualize record relationships.

Tip: Related Info is a sub-view that appears when you select a record in the LifeLog View screen. It is accessible from the LifeLog View.

Definition Management

Element Description
Tabs Each definition is displayed as a tab. Click a tab to switch between definitions.
Definition Name A text field where you can edit the name of the definition.
+ Button (next to tabs) Adds a new definition.
Delete Button Deletes the currently displayed definition.

Query List

Each definition displays a list of queries. Use the floating + button to add a new query.

Query Settings

Setting Type Description
Title Text Input Sets the display name of the query.
Prefix Text Input Text displayed before the matched value.
Suffix Text Input Text displayed after the matched value.
Predicate Group Condition Editor Filters match targets using AND/OR conditions.
Related Time Match Type Dropdown Selects how time-based relevance is determined.
Search Duration (hours) Numeric Input Specifies the time range in hours for searching related records.
Custom Search Query Checkbox + Button Enable to use a custom search query. Click the button to open the search query editor.

Related Time Match Types

Type Description
Near in time Matches the record with the closest time both before and after the target record.
Near in time (before) Matches the record with the closest time before the target record.
Near in time (after) Matches the record with the closest time after the target record.

Query Edit Buttons

Button Description
Reset Reverts the edits to their original state.
Save Saves the query changes.

Bottom Action Buttons

Button Description
Apply Saves all definition and query changes.
Cancel Discards changes and closes the dialog.
Tip: Related information is used to automatically display related records when browsing entries in the life log view.
Tip: A shorter search duration will only match records that are closer in time.
Tip: Create multiple definitions to define different types of relationships.
Tip: Use custom search queries to set up more advanced search conditions.
