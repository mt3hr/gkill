튜토리얼

gkill 사용법을 기본편과 응용편으로 나누어 설명합니다. 처음 사용하시는 분은 기본편부터 순서대로 읽어주세요.

기본편

1. gkill에 로그인하기

최초 실행 시

gkill을 처음 실행하면 계정 등록 화면이 자동으로 표시됩니다.

  1. 사용자 ID를 입력합니다 (이것이 계정 이름이 됩니다).
  2. 비밀번호를 입력하고, 확인을 위해 한 번 더 입력합니다.
  3. 관리자 비밀번호를 설정합니다 (서버 관리에 사용됩니다).
  4. "등록" 버튼을 누르면 계정이 생성됩니다.
  5. 로그인 화면으로 리다이렉트되므로, 생성한 계정으로 로그인합니다.

일반 로그인

  1. 사용자 ID 입력란에 계정의 사용자 ID를 입력합니다.
  2. 비밀번호 입력란에 비밀번호를 입력합니다.
  3. "로그인" 버튼을 클릭하거나 Enter 키를 누릅니다.
힌트: 로그인 후 세션은 30일간 유효합니다. 기한이 만료될 때까지 재로그인이 필요 없습니다.

2. 기록 추가하기

gkill에서는 다양한 종류의 라이프로그를 기록할 수 있습니다. 화면 오른쪽 하단의 "+" 버튼으로 기록을 추가합니다.

기록 유형용도입력 예
메모텍스트 메모를 남기기"오늘은 날씨가 좋았다"
타임스탬프시간 기록 (시작/종료)"회의 10:00~11:30"
태스크해야 할 일을 관리하기"보고서 작성하기"
지출지출 내역을 기록하기"편의점: 삼각김밥 1500원"
기분기분을 0~10 수치로 기록"7"
수치수치 데이터를 기록하기"체중: 65.5"
북마크URL을 저장하기"https://example.com"
파일이미지나 파일 업로드사진, PDF 등

기록 추가 절차

  1. 화면 오른쪽 하단의 "+" 버튼을 탭합니다.
  2. 추가하려는 기록 유형을 선택합니다.
  3. 입력 폼에 내용을 기입합니다.
  4. "저장" 버튼을 눌러 기록을 저장합니다.
힌트: 메모장 화면을 사용하면 여러 기록을 텍스트로 한꺼번에 일괄 입력할 수 있습니다. 자세한 내용은 이후의 "3. 메모장으로 여러 기록을 한꺼번에 입력하기"를 참조하세요.

3. 메모장으로 여러 기록을 한꺼번에 입력하기

메모장 화면에서는 텍스트를 작성하는 것만으로 여러 기록을 한 번에 생성할 수 있습니다.

기본 사용법

  1. 메모장 화면을 엽니다 ("+" 버튼 메뉴에서 "메모장" 선택).
  2. 텍스트 영역에 기록하고 싶은 내용을 입력합니다.
  3. "저장" 버튼 (！을 입력하여 전송도 가능)을 누릅니다.

템플릿을 사용한 입력

템플릿을 사용하면 정형문을 원터치로 입력할 수 있습니다.

  1. 메모장 화면에서 템플릿 목록에서 템플릿을 선택합니다.
  2. 템플릿 내용이 텍스트 영역에 삽입됩니다.
  3. 필요에 따라 내용을 편집하고 저장합니다.
힌트: 자주 사용하는 기록 패턴을 미리 템플릿으로 등록해 두면 편리합니다. 설정 화면의 "템플릿 구조"에서 관리할 수 있습니다.

4. 기록 확인 및 검색하기

라이프로그 뷰 화면에서 과거 기록을 목록 표시하고 검색할 수 있습니다.

기본 조작

  1. 화면 상단 내비게이션에서 라이프로그 뷰 화면을 엽니다.
  2. 기록이 날짜순으로 목록 표시됩니다.
  3. 날짜를 변경하여 특정 날의 기록을 확인할 수 있습니다.

검색과 필터링

  • 태그로 필터링: 특정 태그가 붙은 기록만 표시할 수 있습니다.
  • 날짜 범위: 시작일과 종료일을 지정하여 기간을 좁힐 수 있습니다.
  • 텍스트 검색: 키워드를 포함하는 기록을 검색할 수 있습니다.
힌트: 설정 화면의 "라이프로그 뷰 표시 일수"를 줄이면 로딩이 빨라집니다.

5. 태스크 관리하기

태스크 화면에서는 칸반 보드 형식으로 태스크를 관리할 수 있습니다.

태스크 생성

  1. 태스크 화면을 엽니다.
  2. "+" 버튼으로 태스크를 추가합니다.
  3. 제목, 보드 이름, 기한 등을 입력합니다.
  4. 저장하면 지정한 보드에 태스크가 표시됩니다.

태스크 완료

  1. 완료한 태스크를 탭하여 상세 정보를 엽니다.
  2. 체크 표시를 하여 태스크를 완료 처리합니다.
힌트: 보드를 여러 개 만들어 "업무", "개인", "쇼핑" 등 카테고리별로 관리하면 정리하기 쉬워집니다.

6. 설정 변경하기

설정 화면에서 애플리케이션의 동작을 커스터마이즈할 수 있습니다.

설정 화면 열기

  1. 화면 상단 툴바에 있는 톱니바퀴 아이콘을 탭합니다.
  2. 설정 화면이 열립니다.

자주 사용하는 설정

설정 항목설명
언어UI 표시 언어를 변경합니다.
다크 테마어두운 색상으로 전환합니다.
시작 화면로그인 후 처음 표시할 페이지를 선택합니다.
라이프로그 뷰 이미지 열 수이미지 썸네일의 열 수를 조정합니다.
힌트: 설정을 변경한 후 "적용" 버튼을 눌러주세요. "취소"를 누르면 변경 사항이 폐기됩니다.

응용편

7. 메모장 구문 활용하기

메모장 구문을 사용하면 텍스트 행의 시작 부분에 접두사를 붙이는 것만으로 다양한 종류의 기록을 일괄 입력할 수 있습니다.

8. 템플릿 활용하기

자주 사용하는 메모장 구문 패턴을 템플릿으로 저장해 두면 매번 입력하는 수고를 줄일 수 있습니다.

템플릿 관리

  1. 설정 화면을 엽니다.
  2. "템플릿 구조" 버튼을 클릭합니다.
  3. 템플릿의 추가, 편집, 삭제가 가능합니다.

템플릿 작성 예

예를 들어, 매일의 일기 템플릿을 작성하는 경우:

。일기
、
ーら

이 템플릿을 사용하면 메모장 화면에서 원터치로 일기 양식이 입력되어, 메모 내용과 기분 수치만 추가하면 기록할 수 있습니다.

힌트: 폴더를 사용하여 템플릿을 카테고리별로 정리하면 찾기 쉬워집니다.

9. 타임스탬프로 시간 기록하기

타임스탬프 기능을 사용하면 작업 시간이나 활동 시간을 정확하게 기록할 수 있습니다.

타임스탬프 기본 조작

  1. 시작: 메모장에서 ーた 접두사를 사용하거나, "+" 메뉴에서 타임스탬프를 추가합니다. 제목과 시작 시각을 지정합니다.
  2. 종료: 메모장에서 ーえ 접두사를 사용하거나, 타임스탬프 편집 화면에서 종료 시각을 입력합니다. 제목이 일치하는 타임스탬프가 종료됩니다.

진행 중인 타임스탬프 관리

타임스탬프 목록 화면에서 현재 진행 중인 타임스탬프를 확인할 수 있습니다.

  • 진행 중인 타임스탬프가 목록으로 표시됩니다.
  • 탭하여 종료 시각을 설정할 수 있습니다.
힌트: 메모장 구문의 ーいえ를 사용하면 일치하는 타임스탬프가 없어도 오류가 발생하지 않습니다. 존재 확인 없이 안전하게 종료할 수 있습니다.

10. 집계 뷰로 데이터 분석하기

집계 뷰 화면에서는 기록한 데이터를 그래프나 집계표로 시각화할 수 있습니다.

집계 뷰 사용법

  1. 내비게이션에서 집계 뷰 화면을 엽니다.
  2. 집계 종류나 기간을 선택합니다.
  3. 데이터가 그래프나 표로 표시됩니다.

집계 설정

설정 화면의 "집계 뷰" 버튼에서 집계 방법이나 뷰 구성을 관리할 수 있습니다.

힌트: 기분 수치나 지출 데이터의 추이를 시계열로 확인하면 생활 패턴을 파악할 수 있습니다.

11. 외부 연동 활용하기

북마클릿으로 URL 기록

브라우저의 북마크 바에 북마클릿을 등록하면, 열람 중인 웹 페이지를 원클릭으로 gkill에 저장할 수 있습니다.

  1. 설정 화면의 "북마클릿 주소"에 표시된 URL을 복사합니다.
  2. 브라우저의 북마크 바에 새 북마크로 등록합니다.
  3. 저장하고 싶은 웹 페이지를 열고 있을 때, 등록한 북마크를 클릭합니다.

GPS 기록과 지도 표시

GPS 로그를 기록하고 있는 경우, 지도 위에 이동 이력을 표시할 수 있습니다.

  1. 설정 화면에서 "Google Maps API 키"를 설정합니다.
  2. GPS 데이터가 기록되어 있으면, 라이프로그 뷰에서 지도 표시를 이용할 수 있습니다.
힌트: Google Maps API 키는 Google Cloud Console에서 Maps JavaScript API를 활성화하여 취득하세요.

12. 데이터 백업과 관리

서버 설정 (관리자용)

관리자 계정으로 로그인하면, 설정 화면에 "서버 설정" 버튼이 표시됩니다.

  • 포트 번호: 서버가 수신 대기하는 포트를 변경할 수 있습니다 (기본값: 9999).
  • TLS 설정: HTTPS 통신을 활성화할 수 있습니다.
  • 접근 제한: 로컬 접근만 허용하도록 제한할 수 있습니다.

계정 관리

관리자는 "계정 관리" 화면에서 사용자 계정의 추가 및 삭제가 가능합니다.

데이터 백업

gkill의 데이터는 모두 홈 디렉터리 (기본값: ~/gkill)에 저장되어 있습니다. 이 디렉터리를 백업하면 모든 데이터를 보존할 수 있습니다.

힌트: 정기적으로 홈 디렉터리를 백업하는 것을 권장합니다. 특히 datas/와 configs/ 디렉터리가 중요합니다.
