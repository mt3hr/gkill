Aggregationsansicht

Bildschirm zur Aggregation und Visualisierung von Lebensprotokoll-Daten basierend auf Bedingungen. Es können zwei Typen definiert werden: „Elemente" für einzelne Aggregatwerte und „Listen" für gruppierte Aggregationen nach Schlüssel.

Tipp: Die Aggregationsansicht ist eine Umschaltfunktion innerhalb der LifeLog-Ansicht. Klicken Sie auf das Aggregationssymbol in der Symbolleiste der LifeLog-Ansicht, um sie anzuzeigen.

Definitionsverwaltung

Element Beschreibung
Registerkarten Jede Aggregationsdefinition wird als Registerkarte angezeigt. Klicken Sie auf eine Registerkarte, um zwischen Definitionen zu wechseln.
Definitionsname Textfeld zum Bearbeiten des Namens der Aggregationsdefinition.
＋-Schaltfläche (neben Registerkarten) Fügt eine neue Aggregationsdefinition hinzu.
Löschen-Schaltfläche Löscht die aktuell angezeigte Aggregationsdefinition.

Bildschirmelemente

Element Beschreibung
Datumsbereich Zeigt den Zeitraum für die Aggregation an.
Download-Schaltfläche Lädt die Aggregationsergebnisse im JSON-Format herunter.
＋-Schaltfläche (schwebend) Zeigt ein Menü zur Auswahl zwischen Element oder Liste hinzufügen an.

Elemente (einzelner Aggregatwert)

Zeigt Aggregatwerte der Daten, die den Bedingungen entsprechen, im Format: „Titel: Präfix [Aggregatergebnis] Suffix" an.

Einstellung Typ Beschreibung
Titel Texteingabe Legt den Anzeigenamen des Elements fest.
Präfix Texteingabe Text, der vor dem Aggregatwert angezeigt wird.
Suffix Texteingabe Text, der nach dem Aggregatwert angezeigt wird.
Aggregationsziel Dropdown Wählt die Aggregationsmethode (Anzahl, Summe, Durchschnitt, Max, Min usw.).
Bedingungsgruppe Bedingungseditor Filtert die Aggregationsziele mithilfe von AND/OR-Bedingungen.

Listen (gruppierte Aggregation)

Zeigt Aggregationsergebnisse gruppiert nach Schlüssel in Listenform an.

Einstellung Typ Beschreibung
Titel / Präfix / Suffix Texteingabe Gleiche Einstellungen wie bei Elementen.
Aggregationsziel Dropdown Wählt die Aggregationsmethode.
Gruppierungsschlüssel Dropdown Wählt das Gruppierungskriterium (Titel, Datentyp, Tag, Monat, Wochentag, Datum usw.).
Bedingungsgruppe Bedingungseditor Filtert die Aggregationsziele mithilfe von AND/OR-Bedingungen.

Bedingungsgruppe bearbeiten

Element Beschreibung
Verknüpfungslogik Wählen Sie AND / OR / NOT, um festzulegen, wie die Bedingungen kombiniert werden.
Bedingung hinzufügen Fügt eine neue Bedingung zur Gruppe hinzu.
Gruppe hinzufügen Fügt eine verschachtelte Bedingungsgruppe hinzu.
Gruppe löschen Löscht diese Bedingungsgruppe.
Jede Bedingung Legt den Bedingungstyp und -wert fest (Filtern nach Datentyp, Tag, Titel, Betrag, Stimmungswert usw.).

Operationen

Operation Beschreibung
Doppelklick Öffnet den Editor für das Element oder die Liste.
Rechtsklick Zeigt das Kontextmenü an (Bearbeiten / Löschen).
Drag & Drop Ändert die Anzeigereihenfolge von Elementen und Listen.

Untere Aktionsschaltflächen

Schaltfläche Beschreibung
Übernehmen Speichert alle Definitionsänderungen.
Abbrechen Verwirft die Änderungen und schließt den Dialog.
Tipp: Elemente sind nützlich für einzelne Aggregatwerte (z.B. monatliche Gesamtausgaben), während Listen für gruppierte Aggregationen nützlich sind (z.B. Ausgaben nach Geschäft).
Tipp: Kombinieren Sie Bedingungen, um komplexe Aggregationsabfragen zu erstellen.
Tipp: Verwenden Sie Drag & Drop, um die Anzeige von Elementen und Listen frei neu zu ordnen.
Tipp: Aggregationsergebnisse können als JSON heruntergeladen werden, um sie mit externen Tools zu analysieren.
