# gkill ユーザ向け導入資料

> **注意:** 本文書は開発者・管理者向けの詳細資料です。初めてのユーザーはアプリ内ヘルプの「はじめに」をご覧ください。

## 1. gkillとは

gkillは**ライフログアプリケーション**です。日々のできごとを記録し、後から見返すことができます。

「都度、今の情報を細切れに記録する → gkillが時系列で表示 → 懐かしむ、あるいは自省する」

スタンドアロンで動作し、外部サーバーとの通信は行いません（地図表示のみ例外）。記録データはすべてローカルに保存されます。

### 記録できるもの

| 記録種別 | 用途 | 例 |
|---|---|---|
| テキストメモ（kmemo） | 自由なテキストメモ | 「今日は天気がいい」 |
| 打刻（timeis） | いつからいつまで何をしていたか | 「会社にいた」「カラオケ中」 |
| 気分（lantana） | 今の気分を10段階評価 | 気分: 7/10 |
| 数値（kc） | 数値の記録 | 体重: 65.2, 腹筋: 30回 |
| 支出（nlog） | お金の出入り | 昼食: 800円 |
| ブックマーク（urlog） | ウェブサイトメモ | 気になった記事のURL |
| タスク（mi） | 軽量TODO | 「牛乳を買う」 |
| ファイル（idf_kyou） | 画像・動画・ファイル | 写真、スクリーンショット |

### 補佐機能

| 機能 | 用途 |
|---|---|
| タグ | 記録にタグを付けて整理・検索を補助 |
| テキスト | 記録に補足テキストを付与 |
| 集計 | 数値・支出・件数・気分などの集計（カスタマイズ可能） |
| 関連情報 | 直近の記録や関連する打刻の表示 |
| 定型文 | よく使う記録をボタン1つで入力可能 |

### 特長：「状況」から検索できる

打刻記録を使って、「どこで何をしていたか」という状況から記録を検索できます。

- 「友人Aと東京のカラオケに居た時の写真」
- 「会社にいる時の平均気分」

## 2. インストール方法

### Windows

1. [gkillダウンロードページ](https://github.com/mt3hr/gkill/releases/latest)からAssetsの「windows_amd64_gkill_vX.X.X.zip」をダウンロード
2. ZIPを解凍して「gkill.exe」を実行
3. 初回起動には時間がかかります（データベース初期化のため）

**体験版（サンプルデータ付き）:**
「gkill_sample_data_vX.X.X.zip」をダウンロード・展開し、`LAUNCH_GKILL_SAMPLE_DATA.bat`を起動すると、サンプルデータ入りで試せます。

### Android

1. Assetsから「gkill_vX.X.X.apk」をダウンロード
2. インストール（「詳細」からインストールを許可）
3. ホーム画面から起動

**注意:** インストール時に警告が出ますが、古いAndroid APIを使用しているためです。

### サーバーモード（上級者向け）

1. お使いの環境に合ったバイナリをダウンロード（linux_amd64等）
2. 「gkill_server」を実行
3. ブラウザで `http://localhost:9999` にアクセス
4. ブラウザからPWAとしてインストール可能

**サーバーモードのメリット:**
- PWA依存機能の安定動作
- プッシュ通知の確実性
- キャッシュ制御やアプリ起動時間の短縮

### Wear OS

コンパニオンアプリ（スマートフォン側）とウォッチアプリ（腕時計側）の2つをインストールします。KFTL形式でのクイック記録が可能です。

## 3. 初回起動〜パスワード設定

### 初回起動の流れ

1. gkillを起動すると「初回アカウント登録画面」が表示されます
2. 管理用アカウント（admin）はサーバ初回起動時に自動作成されます。この画面では普段使うアカウントを登録します
3. 以下を入力してください:
   - **ユーザーID** — ログインに使用するアカウント名
   - **パスワード** — ログイン用パスワード
   - **管理用パスワード** — サーバー設定変更時に使用（adminアカウントのパスワード）
4. 「登録」ボタンを押すとログイン画面に遷移します
5. 登録したユーザーID・パスワードでログインしてください

### ログイン後

ログインすると**ライフログビュー画面（RYKV）**が表示されます（設定でデフォルト画面を変更可能）。右下のフローティングボタン（+）から記録を始められます。

## 4. 基本操作

### 記録の追加

**方法1: さいはて画面（Saihate）のFABボタン**

右下の「+」ボタンを押すと、記録種別を選択できます。

| ボタン | 追加される記録 |
|---|---|
| メモ帳（KFTL） | テキスト形式の一括入力画面 |
| 打刻メモ帳（MKFL） | タイムスタンプ付きメモ入力 |
| 数値記録（KC） | 数値記録 |
| ブックマーク（Urlog） | ブックマーク |
| 打刻帳（TimeIs） | 打刻（時間記録） |
| タスク（Mi） | タスク |
| 支出（Nlog） | 支出記録 |
| 気分（Lantana） | 気分値 |

**方法2: メモ帳画面（KFTL）**

メモ帳（KFTL — gkill独自のテキスト書式）を使って、複数の記録を一度にテキストで入力できます。定型文テンプレートをボタンで挿入できるため、よく使う記録パターンを素早く入力可能です。日本語プレフィックス（。、？、ーー等）と ASCII プレフィックス（#, ?, --, /mi, /mood 等）の両方が使えます。

### 記録の編集

1. 記録を表示し、コンテキストメニュー（右クリックまたは長押し）を開く
2. 「編集」を選択
3. 編集ダイアログで内容を変更
4. 保存

### 記録の削除

コンテキストメニューから「削除」を選択します。削除確認ダイアログが表示されます。

### 記録の検索

ライフログビュー画面（RYKV）で以下のフィルタを組み合わせて検索できます。

| フィルタ | 説明 |
|---|---|
| キーワード | テキスト全文検索 |
| 期間 | 日付範囲の指定 |
| タグ | タグでの絞り込み |
| 打刻 | 打刻期間（「会社にいた時」等）での絞り込み |
| リポジトリ | データの格納先での絞り込み |
| 地図 | GPS位置での絞り込み |

### タグの付与

記録にタグを付けることで、後から検索しやすくなります。

1. 記録のコンテキストメニューから「タグ追加」を選択
2. タグ名を入力
3. 保存

## 5. 各画面の使い方

### メモ帳画面（KFTL, `/kftl`）

gkill独自のテキスト書式で記録を一括入力する画面です。

- テキストエリアにKFTL書式で入力
- テンプレートボタンで定型文を挿入可能
- 「送信」で記録が保存される
- 複数種類の記録を一度に入力可能

### タスク画面（Mi, `/mi`）— タスクボード

タスク管理の画面です。

- ボード別にタスクを表示
- タスクのチェック（完了/未完了）
- ボードの切替、ソート順の変更
- 完了状態でのフィルタリング

### ライフログビュー画面（RYKV, `/rykv`）— 履歴閲覧

全記録を時系列で閲覧・検索する画面です。

- 左側のサイドバーで検索条件を設定
- キーワード、期間、タグ、打刻、リポジトリ、地図での絞り込み
- ファイルアップロードも可能

### 記録画面（Kyou, `/kyou`）— 記録詳細

個別の記録を詳細表示する画面です。

- 記録の本体データ
- 付帯情報（タグ、テキスト、通知）
- 関連する打刻情報
- コンテキストメニューから編集・削除

### 打刻メモ帳画面（MKFL, `/mkfl`）

タイムスタンプ付きでメモを入力する画面です。

- タイムスタンプ付きメモの入力
- ファイルのアップロード
- アップロードしたファイルへの日時関連付け

### 実行中画面（Plaing, `/plaing`）— 打刻一覧

現在アクティブな打刻セッションの一覧です。

- リアルタイムのタイマー表示
- 打刻の終了操作

### さいはて画面（Saihate, `/saihate`）— ダッシュボード

メインのダッシュボード画面です。

- 各画面へのナビゲーション
- FABボタンから各種記録の素早い追加
- リロード、ログアウト操作

## 6. 設定画面

サーバー設定やアプリケーション設定は、各画面のメニューからアクセスできます。

### リポジトリ管理

記録データの格納先（リポジトリ）を管理します。

- リポジトリの追加・削除・有効化/無効化
- RepType（データ型）の指定
- 書き込み先リポジトリの指定

### テンプレート管理

KFTL画面で使用する定型文テンプレートを管理します。

- テンプレートの追加・編集・削除
- テンプレート構造の階層管理

### タグ構造管理

タグの階層構造を管理します。

- タグカテゴリの追加・編集
- タグの親子関係の設定

### デバイス管理

使用するデバイスの管理を行います。

- デバイスの追加・編集・削除
- デバイスごとのリポジトリ割当

## 7. よくある問題と解決方法

### 起動に関する問題

| 症状 | 解決方法 |
|---|---|
| 初回起動が遅い | データベース初期化中です。しばらくお待ちください |
| ポートが使用中 | 他のgkillプロセスが起動中でないか確認してください |
| 画面が表示されない | ブラウザで `http://localhost:9999` に直接アクセスしてみてください |

### 記録に関する問題

| 症状 | 解決方法 |
|---|---|
| 記録が保存されない | ログイン状態を確認してください。セッション有効期限は30日です |
| 検索結果が出ない | フィルタ条件を緩めてみてください。「クリア」ボタンでリセットできます |
| ファイルがアップロードできない | ファイルサイズ制限は設定されていません（デフォルト無制限） |

### 通知に関する問題

| 症状 | 解決方法 |
|---|---|
| 通知が届かない | PWAとしてインストールし、通知許可を有効にしてください |
| 通知の時刻がずれる | 通知機能は環境に依存します。おおよその時刻で通知されます |

### その他

| 症状 | 解決方法 |
|---|---|
| パスワードを忘れた | 管理者にパスワードリセットリンクの発行を依頼してください |
| データを別のPCに移動したい | `$HOME/gkill/`フォルダをコピーしてください |

## 8. バックアップと復元

### バックアップ

gkillの全データは `$HOME/gkill/` ディレクトリに保存されています。このディレクトリを丸ごとコピーすることでバックアップできます。

```
バックアップ対象:
$HOME/gkill/
├── configs/    ← 設定データベース
├── datas/      ← ユーザーデータ
├── caches/     ← キャッシュ（バックアップ不要、再生成可能）
├── logs/       ← ログ（バックアップ不要）
└── tls/        ← TLS証明書（使用している場合）
```

**最低限のバックアップ対象:**
- `configs/` — 全設定情報
- `datas/` — 全記録データ

### 復元

1. gkillを停止
2. バックアップした `$HOME/gkill/` を元の場所にコピー
3. gkillを起動

**注意:** キャッシュは起動時に自動再生成されるため、バックアップに含めなくても問題ありません。

### 定期バックアップの推奨

重要な記録データを保護するため、定期的なバックアップを推奨します。`configs/`と`datas/`ディレクトリをコピーするだけで完了します。
